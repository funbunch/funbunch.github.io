Create content tailored to responsive design

The web browsing experience is fundamentally shifting as users increasingly access sites using a variety of devices. A recent IDC study projected that by 2015, more U.S. consumers will be accessing the internet through mobile devices than through PCs. To adapt to changing user behaviors, designers are implementing responsive design, which tailors page displays to the screen of the viewing device.

More dynamic user interfaces cannot solve all the challenges of personalizing the user experience, however. Content authors must align their work with that of designers and developers to ensure that a site is not only viewable on a range of devices but also offers usable materials to visitors. A recent Content Marketing Institute blog post noted that, to be effective, responsive design requires a comprehensive content strategy that works across different channels.

“The best designs pull input from a multidisciplinary team composed of content strategists, designers, developers and marketing technologists,” Content Marketing Institute contributor Giovanni Calabro wrote.

Developing a content strategy that works smoothly with responsive design requires organizations both to prioritize which items to display and to follow certain guidelines during the content creation process that guarantee flexibility.

Prioritizing content

Mobile users do not only have smaller screens than desktop users; they also may be accessing a site over a network with data constraints. As a result, mobile sites must be more streamlined than desktop sites. The fundamental content consideration for responsive design is what information users on a given device value the most.

“Content prioritization is the first step in a user-centric approach to responsive design,” Calabro explained. “Break your web content into core elements or building blocks. To prioritize, ask yourself, ‘What do my customers need on a smartphone? Tablet? Desktop? How will they use each device to access information?’”

Determining the simplest, most important version of a site’s content and building out from there will ensure that content is as responsive as the graphical layout of a site. In a column for FastCo.Create, web design experts Miller Medeiros and David Vale explained that site design should begin with a bare bones version that offers the most basic set of features and concise messaging possible. Reducing the number of points at which a site can break or become unintuitive is the goal of responsive design in terms of both display and content.

Designing flexible content

Content management systems (CMS) allow organizations to place the responsibility of content authorship in the hands of less technically savvy users. While content creation once had to go through design or development staff - a webmaster - the path from idea to implementation is shorter than ever, which has made work more efficient for all parties involved. According to iMedia Connection contributor Gabe Sumner, the downside of this consolidation is that authors can more easily deploy content that impedes responsive design without realizing what they are doing. For instance, the use of a multi-column table to display information may not scale to all screens.

To avoid constantly creating issues for designers to fix, Sumner suggested organizations can pay attention to certain guidelines that will standardize content authorship. Ideally, he said, a CMS will automatically smooth some issues that cause responsive design headaches, such as offering a rich text editor interface that removes thorny embedded formatting and limits unneeded features such as tables.

Additionally, designers can offer users templates that limit the number of editable regions to a set of predefined fields and give authors layout alternatives that let them create visually rich content without relying on features such as tables that create responsive design problems. Users can then deliver content in formats that are easily reconfigured across different responsive site layouts.
