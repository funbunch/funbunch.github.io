Key considerations for improving SEO in a CMS

Revisions to Google's algorithms have led some marketers to worry that search engine optimization (SEO) techniques are no longer effective for building traffic. While quick tricks such as content duplication, keyword stuffing and semi-hidden text have been punished by Google's Panda and Penguin updates, techniques that legitimately help classify site information are still critical for quality performance in search rankings.

A flood of low-quality content

To compete in a world in which search engines are interested in identifying useful information, fixing technical issues that may obscure that information should be a top priority for any SEO expert. Content management systems (CMS) are designed to help smooth and automate the process of ensuring the technical details surrounding content are in place, but marketers may need to look beyond the most basic features to understand how a CMS can truly improve SEO.

“Using a CMS to build a site doesn't automatically imbue it with SEO magic,” B2B marketer Dave Ross wrote in a blog post for MarketingProfs. “SEO is a process, not a feature, and today's tools make it easy to automate the important (yet repetitive) tasks of SEO management.”

While a basic SEO checklist begins with using features common to a CMS - title tags, meta description tags, meta keyword tags and header tags - marketers must go beyond these functions to see real results, Ross explained. A recent Search Engine Watch column by Ray Comstock identified some other CMS features critical to SEO.

Features to look for in a CMS

Comstock highlighted the importance of using a CMS that populates sites in formats that search engines can read. Drop-down navigation menus, which are critical in indexing a site’s internal link structure for search engines, should be in CSS, for instance, rather than a format such as Flash, as JavaScript that is harder for Google to crawl. Similarly, a CMS should be able to automatically produce an XML sitemap, and it should avoid frames and iframes, which search engines have trouble indexing.

Many of the most common problems related to the revised Google algorithms relate to URLs and pagination. Rather than delivering multiple URLs for the same content or URLs that show no relationship between similar content, a CMS should be able to standardize URL creation in an SEO-friendly way, Comstock explained. This means that it should create static, keyword-focused URLs that use tags such as rel=canonical, rel=prev and rel=next that will be easily searchable and avoid duplicate content. The CMS should also use 301 redirection headers rather than 302 redirects to ensure that link connectivity metrics are transferred in the redirect.

A CMS should also support customization of breadcrumb navigation, Comstock said. Breadcrumb navigation, in which users can see their path through a site, is helpful for both the user experience and link connectivity. It should allow for unique, keyword-focused navigation, but it also should not create unique URL paths. Alt tags are another important point of customization, as they allow search engines to see images hosted on a page and contribute to keyword relevancy.

Ensuring the deeper functionality and customization of a CMS can greatly enhance legitimate SEO practices. Rather than spending their time trying to trick search engines, marketers should be diving into the features of their CMS that allow them to more clearly highlight the existing content on a site.

“A good CMS can have a dramatic impact on your SEO performance and conversely, a bad CMS can kill your SEO program and any hope you have of success,” Comstock concluded. “Take the time to evaluate how yours may be affecting your program.”.
